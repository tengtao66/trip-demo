Key UX Guidelines Applied

Applied to Design

  • Storytelling hero — Emotional headline, not just product listing
  • Payment flow explainer — 3-step visual guide before trip cards
  • Donut over pie — Center total value, direct labels with amounts (no detached legend)
  • Stacked area — Shows trend + composition in one chart
  • KPI progress bars — Bullet-chart style context for each metric
  • WCAG AA contrast — All text meets 4.5:1 minimum on dark surfaces
  • Semantic color tokens — Consistent warm amber (#A16207) for CTA/accent
  • Photography-forward cards — Larger image ratio (60/40 vs 40/60)
  • Simulation mode — Prominent banner for demo fast-forward
  • Step indicators — Multi-step checkout will show progress

Anti-Patterns Avoided

  • No emoji icons — Use Lucide React SVG icons throughout
  • No pie chart >5 slices — Only 3 segments, with donut variant
  • No color-only meaning — All status badges include text labels
  • No placeholder-only labels — All forms use visible labels
  • No hardcoded hex — Semantic tokens via Tailwind CSS
  • No random shadows — Consistent border-based elevation
  • No heavy text hero — Photography + short tagline
  • No hover-only interactions — All actions have tap/click targets
